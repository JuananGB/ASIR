# <span style="color:lightcoral"> 2.6 El modelo relacional </span> 

### El objetivo del modelo relacional es proteger al usuario de la obligación de conocer las estructuras de datos físicas con las que se representa la información de una BBDD. Desvincular estas estructuras de datos, que son complejas por naturaleza, del diseño lógico(Modelo Relacional), permite que la base de datos se pueda implementar en cualquier gestor de bases de datos relacional(Oracle,MySQL,DB2,etc.) .

## <span style="color:lightcoral"> 2.6.2 Otros conceptos del modelo relacional

### A continuación se definen los conceptos necesarios para transformar el modelo conceptual(E/R) en el modelo lógico (modelo relacional)

* ### <span style="color:lightcoral"> **Atributo** </span> :  Características que describen a una entidad o relación.

* ### <span style="color:lightcoral"> **Dominio** </span> : Conjunto de valores permitidos para un atributo.

* ### <span style="color:lightcoral"> **Restricciones de semántica** </span> : Condiciones que deben cumplir los datos para su correcto almacenamiento. Hay varios tipos:
     * ### <span style="color:lightcoral"> Restricciones de clave </span> : Conjunto de atributos que identifican de forma única a una entidad.
     * ### <span style="color:lightcoral"> Restricciones de valor único (UNIQUE) </span> : Restricción que impide que un  atributo tenga un valor repetido. 
     * ### <span style="color:lightcoral"> Restricciones de integridad referencial </span> : Se da cuando una tabla tiene una referencia a algun valor de otra tabla. En este caso la restricción exige que exista el valor referenciado en la otra tabla.
     * ### <span style="color:lightcoral"> Restricciones de dominio </span> : El valor que puede tomar unn campo esté dentro del dominio definido.
     * ### <span style="color:lightcoral"> Restricciones de verificación (CHECK)</span> : Esta restricción permite comprobar si un valor de un atributo es válido conforme a una expresión
     * ### <span style="color:lightcoral"> Restricciones de valor NULO (NULL o NOT NULL) </span> : Un atributo puede ser obligatorio si no admite el valor NULO o NULL , es decir, el valor falta de información o desconocimiento. SI admite como valor el valor NULL, el atributo es opcional.
     * ### <span style="color:lightcoral"> Disparadores o triggers </span> : Son procedimientos, que se ejecutan para hacer una tarea concreta en el momento de insertar, modificar o eliminar informaciñón de una tabla.
     * ### <span style="color:lightcoral"> Restricciones genéricas adicionales o aserciones (ASSERT) </span> : Permite validar cualquiera de los atributos de una o varias tablas .
* ### <span style="color:lightcoral"> **Clave** </span>: Una clave es un conjunto de atributos que identifican de forma única una ocurrencia de entidad.
    * ### <span style="color:lightcoral"> Superclave</span> 
    * ### <span style="color:lightcoral"> Clave Candidata </span>
    * ### <span style="color:lightcoral"> Clave primaria </span> : Es la clave candidata elegida por el diseñador como clave definitiva 
    * ### <span style="color:lightcoral"> Clave foránea </span> : Es un atributo de una entidad, que es clave en otra entidad. Ejemplo (La nota en un módulo de una asinatura corresponde a un DNI, que es clave de otra entidad. En este caso el DNI es una clave foránea en la tabla notas).

# <span style="color:orange">1.6 Lost Gestores de Arranque

### El gestor de arranque es un pequeño programa que se ejecuta una vez completado el inicio normal de la BIOS y que permite seleccionar que sistema operativo en caso de disponer arranque múltiple

### <span style="color:orange"> NTLDR </span>
### Encargado del arranque de los SO (WindowsXP,Windows Server 2003) . Se encuentra normalmente, en el directorio raíz del disco del sistema. Requiere como mínimo que se encuentre en dicho directorio raíz: el archivo boot.ini

### <span style="color:orange"> Bootmgr </span>
### Encargado de los arranques de (Windows7, Windows Server 2008) . Se encuentra en la raíz del disco del sistema .

### Al iniciar el arranque llama al archivo <span style="color:orange"> WinLoad.exe </span> que es el cargador del SO windows (se encuentra en el directorio \Windows\system32) y dará paso al archivo <span style="color:orange"> ntoskrnl.exe </span> que se encargará del resto del arranque del sistema

### <span style="color:orange"> Lilo </span>
### Gestor de arranque de linux que permite iniciar este SO junto con otras plataformas.

### <span style="color:orange"> Grub </span>
### Es más moderno y flexible que Lilo ya que permite que el administrador ejecute cualquier comando desde la linea de comando de Grub. Entre todas sus características hay que destacar la posibilidad de incluir múltiples formatos de ejecutables.
